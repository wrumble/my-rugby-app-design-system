Anatomy

The Match Hub uses its own custom tab bar — not the bottom NavigationBar from the rest of the app. Layout is a Stack over Material(elevation: 3):

  • Side tabs render in two flex rows around a fixed SizedBox(width: 88) centre slot
  • The Fan Zone tab is positioned top: -28 overflowing above the bar — a 64×64 radial-gradient orb (#1A1A2E → AppColors.red500) with BoxShadow(0,5,10, 25%)
  • Selected side tab: red label + 24×2 underline pill (AnimatedContainer 300ms easeInOutCubic)
  • Selected flagship: 3pt red ring around the orb + red label
  • Bar height 64pt; flagship total height 96pt; respects bottom SafeArea

Iconography — unified set. Match Hub icons follow the app-wide line style: 24px viewBox, 1.6pt stroke, strokeLinecap: round, strokeLinejoin: round, no fills (only the fan hub is filled). Selected = stroke #FF6B78; default = stroke #7A8FA3. Use Material Symbols Outlined as the source where possible: sports_rugby (Live) · stadium (Stadium) · tv (Watch) · menu_book (Programme) · local_bar (Pre-Match) · directions_bus (Post-Match). The flagship Fan Zone uses a custom electric-fan glyph (4 curved blades + filled hub + housing ring + stand) — not wind_power, which reads as a turbine. Apply this style to every icon in the app: replace any filled/duotone Material Icons with their _outlined variants, drop legacy emoji on MatchHubTab.icon and NavigationDestination, and align tab-bar / app-bar / list icons to 22pt with the same 1.6pt outlined stroke.

Tab list selection is driven by tabsForMode(mode, phase:): Ground mode returns 5 tabs (with Pre-Match/Post-Match swap on MatchPhase.fullTime); TV mode returns 3 tabs unconditionally. The centre Fan Zone is hard-wired via fanZoneIndexFor(mode) — index 2 for Ground, index 1 for TV — so each layout is symmetric around the flagship.

Mode entry. First time a user opens a match, ModeSelectionScreen runs a 3-step flow: GPS auto-detect → confirm "you're at the stadium?" → fall-back manual select with two cards comparing what each mode shows. Selection persists per-matchId via ModeBloc; users can swap any time via the app-bar action which dispatches ModeModeSelected.
