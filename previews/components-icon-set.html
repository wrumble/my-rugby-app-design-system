Zero-emoji policy. Every emoji shown above must be replaced with the corresponding line icon. Specifically:

  • MatchHubTab.icon (Dart) — change type from String to IconData; pass Icons.sports_rugby_outlined, Icons.stadium_outlined, etc. The Fan Zone flagship uses a custom FanZoneIcon widget (CustomPainter) — not Icons.wind_power.
  • EventFeedSection — swap the emoji-string event icons for Icon(Icons.sports_rugby_outlined, ...) etc. Yellow/Red cards: filled rectangles with amber / red colour from AppColors.
  • ReactionBar — the only place reactions still render as glyphs, but switch to a custom outlined glyph set (heart, flex-arm, shock, sad, angry) so they match the rest of the iconography. Each reaction keeps its ReactionType enum + analytics key.
  • ModeSelectionScreen — pulsing 📍 emoji becomes Icon(Icons.place_outlined) with the same scale-pulse animation.
  • NoMatchScreen — 🏉 hero glyph becomes a 64pt Icons.sports_rugby_outlined.
  • StadiumTabScreen transport rows — 🚌 🚂 🚗 ☁️ become directions_bus_outlined, train_outlined, directions_car_outlined, cloud_outlined.

Accessibility labels (shared.*_a11y i18n keys) stay attached to every icon — line icons need them more, not less.
